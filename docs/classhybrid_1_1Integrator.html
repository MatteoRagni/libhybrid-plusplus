Detailed Description

template<typename REAL_T, std::size_t dim_x, std::size_t dim_u, std::size_t dim_p, typename... Args>
class hybrid::Integrator< REAL_T, dim_x, dim_u, dim_p, Args >

Abstract class for integrators.

This is an abstract class for integrators that do not use dynamical memory allocation. All dimensions are defined in a static way, through templating. The ode is required to be defined as:

\[ \dot{x}(t) = f(t, x(t), u(t), p), \qquad x(t) \in \mathbb{R}^{\dim(x)}, \, u(t) \in \mathbb{R}^{\dim(u)} \, p \in \mathbb{R}^{\dim(p)} \]

The ODE styles is in same way compatible with the MATLAB System identification toolbox idnlgrey function call. The only difference is in the parameter vector that in this case is in a single array (instead of a REAL_T** vector).

Template Parameters
REAL_Tfloating point precision
dim_xdimension of the state array
dim_udimension of the external forces array
dim_pdimension of the parameters array
Argsvariadic template for user arguments

The method evaluates the next step in the integration.

The method gets as input the current ode value, the current time, the current parameters and the current input and outputs the next time and the next state vector. This function is purely virtual (and in fact Integrator is an abstract class), thus it must be overriden in the final integrator.

Parameters
xpthe array that contains the value of the state at the next time step
tcurrent time. This variable will contains the next time ( \( t^+ = t + t_s\))
xcurrent state vector. This array is constant
ucurrent input vector. This array is constant
pcurrent parameters vector. This array is constant
datauser data space (as variadic argument)

Implemented in hybrid::RK4< REAL_T, dim_x, dim_u, dim_p, Args >, and hybrid::ExplicitEuler< REAL_T, dim_x, dim_u, dim_p, Args >.
